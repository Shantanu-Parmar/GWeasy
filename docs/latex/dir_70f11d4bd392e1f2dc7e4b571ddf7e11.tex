This software aims to reduce the complexity of installing and running GW analysis tools by integrating all necessary software requirements into a single, easy-\/to-\/use platform.\hypertarget{README.md_autotoc_md2}{}\doxysubsubsubsection{\texorpdfstring{{\bfseries{Project Details}}}{{\bfseries{Project Details}}}}\label{README.md_autotoc_md2}

 \hypertarget{README.md_autotoc_md4}{}\doxysubsubsection{\texorpdfstring{{\bfseries{1. Introduction}}}{{\bfseries{1. Introduction}}}}\label{README.md_autotoc_md4}
The {\bfseries{Laser Interferometer Gravitational-\/\+Wave Observatory (LIGO)}} operates {\bfseries{four detectors}} and collects data across {\bfseries{six runs}}, with over {\bfseries{300 channels}}, multiple sampling rates, and countless time segments. This raw time-\/series GW data is then analyzed using various pipelines such as\+:
\begin{DoxyItemize}
\item {\bfseries{OMICRON}}
\item {\bfseries{Coherent Wave\+Burst (c\+WB)}}
\item {\bfseries{Matched filtering}}
\item {\bfseries{Fourier transforms}}
\end{DoxyItemize}

Currently, setting up the software environment for such analyses is a major challenge. The goal of this project is to simplify this process by creating a GUI-\/based tool that can handle everything—from \texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}installation to execution\texorpdfstring{$\ast$}{*}\texorpdfstring{$\ast$}{*}—without requiring extensive technical knowledge.

 \hypertarget{README.md_autotoc_md6}{}\doxysubsubsection{\texorpdfstring{{\bfseries{2. Background \& Motivation}}}{{\bfseries{2. Background \& Motivation}}}}\label{README.md_autotoc_md6}
While working with GW data, I encountered numerous challenges, spending over {\bfseries{10 weeks}} resolving library conflicts, configuring environment files, and understanding poorly documented installation processes. Through collaboration with scientists, developers, and LIGO repository maintainers, I finally managed to get the {\bfseries{correct workflow}} established.

This experience highlighted the need for a {\bfseries{user-\/friendly, platform-\/independent tool}} that can\+:
\begin{DoxyItemize}
\item Automate the {\bfseries{installation}} of required dependencies.
\item Manage {\bfseries{multiple pipelines}} with ease.
\item Provide an {\bfseries{intuitive GUI}} for data selection, analysis, and visualization.
\item Offer a {\bfseries{one-\/click}} install and execution process.
\end{DoxyItemize}

Many astrophysicists spend more time setting up the software than analyzing the data. {\bfseries{\doxylink{namespace_g_weasy}{GWeasy}}} aims to {\bfseries{eliminate this technical barrier}}, allowing researchers to focus on their primary goal\+: {\bfseries{gravitational wave science}}.

To perform a test run\+:
\begin{DoxyItemize}
\item Select {\bfseries{{\ttfamily test.\+csv}}} as the time segment input.
\item Select {\bfseries{{\ttfamily 4\+KCHANS.\+csv}}} as the channels input.
\item Optionally, specify an output directory. By default, GWEasy will create a folder named {\ttfamily gwfout} in the same directory.

\hypertarget{README.md_autotoc_md18}{}\doxysubsubsubsubsection{\texorpdfstring{Configuring Omicron Settings}{Configuring Omicron Settings}}\label{README.md_autotoc_md18}
If you have worked with {\bfseries{Omicron configuration files}}, this setup will feel familiar\+:



{\bfseries{Steps\+:}}
\begin{DoxyEnumerate}
\item {\bfseries{Set Channel \& Sampling Rate First}} (due to an existing UI issue, these must be configured before modifying other fields).
\item Channels are pre-\/populated based on the {\bfseries{Gravfetch}} output, but you can also manually enter any channel name.
\end{DoxyEnumerate}




\begin{DoxyEnumerate}
\item {\bfseries{Select the {\ttfamily .ffl} files}} corresponding to each channel. These are located in the {\bfseries{{\ttfamily gwfout}}} directory or your chosen output path. Each channel has an {\ttfamily .ffl} file listing all time segments.
\item Use the {\bfseries{Custom Segments}} feature to manually specify time segments for analysis if needed.
\end{DoxyEnumerate}




\begin{DoxyEnumerate}
\item Once all fields are filled, specify an output directory (default\+: {\ttfamily Omicronout}).
\item Click {\bfseries{Save Config}} to generate a configuration file with the selected inputs.
\end{DoxyEnumerate}




\begin{DoxyEnumerate}
\item Press {\bfseries{Start Omicron}} to begin processing. Upon successful execution, a completion message will be displayed in the terminal.
\end{DoxyEnumerate}
